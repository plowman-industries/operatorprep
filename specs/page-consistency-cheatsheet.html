Page Consistency Cheat Sheet

The one-pager. Paste this into any AI session where you're editing a practice test, flashcard, math drill, or study guide landing page. Prevents the "every update changes the structure" problem.

The Problem We're Solving

Every grade-specific landing page (60 total: practice tests, flashcards, math drills, and study guides for T1-T5, D1-D5, WW1-WW5) must have identical structure. When pages get rebuilt, the mastery legend disappears, the hero card changes, or the divider copy gets rewritten. This sheet locks that down.

Practice Test Landing Page — Required Anatomy

// In this exact order

  1. Page header — "[Grade] Practice Tests" + "Select a category to start a random 25-question quiz"
  2. Mastery legend — 4 dots: Mastered, Review, Needs Work, Not Attempted
  3. Full practice hero card — 25 random from all N categories
  4. Divider label — "OR FOCUS ON A SPECIFIC TOPIC"
  5. Topic grid — 4 cols x 2 rows, 8 topic cards with mastery stripes

Flashcards Landing — Same Anatomy, Different Copy

// Identical structure to practice tests

  1. Page header — "[Grade] Flashcards"
  2. Mastery legend — same 4 dots
  3. "Review due" hero card — "N cards due for review today"
  4. Divider label — "OR STUDY BY TOPIC"
  5. Topic grid — same 8 topics, same layout

Math Drills Landing — Same Anatomy, Different Copy

// Identical structure

  1. Page header — "[Grade] Math Drills"
  2. Mastery legend — same 4 dots
  3. Mixed drill hero card — "10 random problems, all formula areas"
  4. Divider label — "OR DRILL BY FORMULA"
  5. Formula grid — 4 cols, dosing / flow / detention / etc.

Study Guide Landing — The One Exception

// No mastery legend — uses sequential progress instead

  1. Page header — "[Grade] Study Guide"
  2. Progress bar — "Chapter X of Y · ~N hrs remaining"
  3. Resume reading hero — jumps to last-read chapter
  4. Divider label — "OR BROWSE CHAPTERS"
  5. Chapter list — sequential, checkmarks for completed

Mastery Legend — Exact Specification

95%+ Mastered 85-94% Review Below 85% Needs Work Not Attempted

The thresholds (95%, 85%) and the dot colors are locked. The copy is locked. Don't change "Below 85% Needs Work" to "Below 85%" or "Needs Improvement" or anything else.

Locked Copy — Don't Paraphrase

Subtitle: "Select a category to start a random 25-question quiz"
Divider (practice): "OR FOCUS ON A SPECIFIC TOPIC"
Divider (flashcards): "OR STUDY BY TOPIC"
Divider (math): "OR DRILL BY FORMULA"
Divider (study guide): "OR BROWSE CHAPTERS"
Legend labels: "95%+ Mastered" · "85-94% Review" · "Below 85% Needs Work" · "Not Attempted"

The AI Preamble — Copy & Paste

Before asking any AI to edit a practice-test, flashcard, math-drill, or study-guide page, paste the design system v1.2 + brand guidelines, and lead with this:

Here is the OperatorPrep design system v1.2 and brand guidelines v1.0. RULES — non-negotiable, violations will be rejected: 1. Do not modify design tokens (colors, spacing, fonts). 2. Do not add icons outside the sprite sheet. Do not use emojis. Do not use Font Awesome. 3. Do not restructure page templates. The mastery legend, hero card, divider, and topic grid on practice-test/flashcard/math-drill pages are REQUIRED and must appear in the order defined in Section 04 of the design system. 4. Do not invent new component variants. Reuse what exists. 5. Do not paraphrase locked copy: "OR FOCUS ON A SPECIFIC TOPIC", "Select a category to start a random 25-question quiz", "95%+ Mastered", "85-94% Review", "Below 85% Needs Work", "Not Attempted". Change ONLY the following: [describe the specific change] Leave everything else identical to what is in the design system.

When Something Really Needs to Change

The process — no shortcuts

  • Bump the design system version first (v1.2 → v1.3).
  • Add the change to the changelog block at the top of the CSS.
  • Update the affected template's anatomy section.
  • Keep the previous version as a backup file.
  • Then apply the change to pages.
